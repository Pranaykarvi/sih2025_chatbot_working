# ArogyaLink ‚Äî `chatbot_module_online`

This folder contains the **production-style** stack: **FastAPI** backend, **Next.js** frontend (`frontend_latest`), Docker assets, and pinned Python dependencies. For the **full project story** (architecture diagrams, hosting, API tables), start with the **[root `README.md`](../README.md)**.

---

## What lives here

| Path | Description |
|------|-------------|
| `chatbot-backend/` | **FastAPI** app: `/embed/pdf`, `/chat/ask`, `/sync/ingest`, `/health`. |
| `frontend_latest/` | **Next.js 14** UI (ArogyaLink): chat + upload; calls the API via `NEXT_PUBLIC_API_URL`. |
| `requirements.txt` | **Direct** dependency pins (FastAPI 0.115.x, `google-genai`, `groq==1.1.2`, `httpx==0.28.1`, Cohere, Supabase, ‚Ä¶). |
| `requirements.lock.txt` | **Full** `pip freeze` output for reproducible installs. |
| `Dockerfile` | Python 3.11 image: `pip install --no-cache-dir -r requirements.txt`, then `uvicorn app.main:app`. |
| `docker-compose.yml` | Runs the backend container with a local `.env`. |

Older or alternate UIs: `chatbot-frontend/`, `chatbot-frontend-new/` (see root README).

## Backend setup (quick)

1. **Supabase:** run `chatbot-backend/app/models/sql/patient_documents.sql` in the SQL editor (`pgvector` + `match_patient_documents`).
2. **Environment:** create `chatbot-backend/.env` (or project-root `.env` for Docker) with at least:
   - `COHERE_API_KEY`, `SUPABASE_URL`, `SUPABASE_KEY`
   - `GEMINI_API_KEY`, `GROQ_API_KEY`
   - Optional: `VECTOR_DIM=1024` (must match DB and Cohere embedding size)
3. **Install & run:**

   ```bash
   pip install -r requirements.txt
   cd chatbot-backend
   uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
   ```

4. **Check:** [http://localhost:8000/health](http://localhost:8000/health) ‚Üí `{"status":"ok"}`.

**LLM behavior:** Gemini first, Groq mandatory fallback, structured logging (`Trying provider=‚Ä¶`, `FINAL RESPONSE GENERATED BY: ‚Ä¶`). See root README section *LLM orchestration*.

**Dependency rule:** keep **`httpx>=0.28.1`** (required by `google-genai`) and a **current `groq`** (e.g. **1.1.2**) so the Groq client does not crash on `httpx` 0.28‚Äôs removal of `Client(proxies=...)`.

## Frontend setup (quick)

```bash
cd frontend_latest
npm install
```

Create `frontend_latest/.env.local`:

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000), set Patient ID (e.g. **`patient0021`**), upload PDFs, then chat. Client uses **180s** timeouts and shows a **cold-start hint** after ~6s while waiting.
